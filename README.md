# rainbow-landing

Marketing site for [Rainbow](https://askrainbow.ai) — the AI sales assistant for field sales reps.

Not the app. The app lives at [app.askrainbow.ai](https://app.askrainbow.ai).

## Stack

- Next.js 16 (App Router, SSG)
- React 19
- TypeScript
- Tailwind CSS v4
- Instrument Serif + Inter + JetBrains Mono (via `next/font`)
- Deployed to Vercel
- Zero client-side data fetching — every byte is pre-rendered for AEO

## Structure

```
src/
├── app/
│   ├── layout.tsx      # Root layout, fonts, metadata, JSON-LD injection
│   ├── page.tsx        # Homepage composition
│   ├── globals.css     # Rainbow theme tokens
│   ├── llms.txt/       # /llms.txt route for LLM crawlers
│   ├── robots.ts
│   └── sitemap.ts
├── components/
│   ├── brand/          # Arc, Logo, RainbowDivider, RInk, ArcBullet
│   ├── hero/           # Animated chat (SpectralWave + Typewriter + IPhoneFrame)
│   ├── sections/       # Nav, Hero, Problem, ProductExplain, Features, Testimonials, FAQ, CTA, Footer
│   └── seo/            # JsonLd injector
└── content/
    ├── direction.ts    # All copy — single source of truth
    └── schemas.ts      # JSON-LD schema builders
```

## Development

```bash
npm run dev     # http://localhost:3000
npm run build   # Prod build + SSG check
npm run lint
```

## Editing copy

All page copy lives in `src/content/direction.ts`. Change a string, the page changes.

FAQ, claims, and comparison snippets also in that file. JSON-LD schema auto-regenerates from the same data.

## SEO / AEO

- Meta + OG + Twitter cards → `src/app/layout.tsx`
- JSON-LD (SoftwareApplication, Organization, FAQPage, BreadcrumbList) → `src/components/seo/JsonLd.tsx`
- `/llms.txt` — short markdown summary for LLM crawlers → `src/app/llms.txt/route.ts`
- `/sitemap.xml`, `/robots.txt` — auto-generated by Next.js

## Deploy

Pushes to `main` auto-deploy via Vercel.

Preview deploys run on every PR.
